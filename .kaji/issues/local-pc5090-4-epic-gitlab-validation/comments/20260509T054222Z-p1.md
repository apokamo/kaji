## OQ-2 決定文書（`kaji-pr-mr-bridge.md`）レビュー指摘への対応報告

### 対応済み (2 件)

#### MF-1: `review --approve` / `--request-changes` の body 取り扱い

**指摘**: `glab mr revoke` 単体では body を投稿できず、既存 `pr-verify` の `--body-file` 付き正式レビューと矛盾する。

**修正内容**:

1. Tier B 表の `review` 行を 2 行に分離（`--approve --body[-file]` と `--request-changes --body[-file]`）
2. `glab mr note --message <body>` + `approve` / `revoke` の **シーケンス実行** として contract 化
3. 新規節「**body 取り扱い原則**」を追加し以下を明記:
   - body は必ず note として記録（捨てない）
   - 順序は「note 投稿 → approve / revoke」（投稿失敗時の不整合最小化）
   - `--request-changes` で未 approve 状態なら revoke は no-op として skip し note のみ実施
   - skill 側 contract 上は GitHub と同様に「body 付き review が成立した」状態を返す
4. GitLab approval state と GitHub review state の非対称性に関する設計含意を補足（確定事項 #7 と同様 provider 内部で吸収）

#### MF-2: `kaji pr merge` の no-ff / squash 禁止契約

**指摘**: `glab mr merge` には GitHub `--merge` 相当の明示 flag がなく、merge method は GitLab project 設定依存。「ほぼ素通し」では弱い。

**修正内容**:

1. Tier B 表の `merge` 行を強化: 「`--squash` / `--rebase` flag を kaji 側で拒否（CLAUDE.md "Merge: --no-ff only" 遵守）」を明記
2. 新規節「**merge method 保証範囲**」を追加し以下を明記:
   - **kaji が保証する範囲**: `--squash` / `--rebase` を kaji 経由で渡せない、`glab mr merge` 呼び出し時にこれら flag を付加しない
   - **GitLab project 設定への依存範囲**: Merge method（`Merge commit` / `Fast-forward` 等）と Squash 設定（`Do not allow` ～ `Require`）が運用と矛盾する場合の挙動を明示
   - **mitigation**:
     - **(ii)** `docs/cli-guides/gitlab-mode.md` に必須前提を記載（Merge method = `Merge commit`、Squash = `Do not allow` または `Allow`）→ 子 Issue #5 の責務
     - **(iii)** 将来の preflight 統合候補として記録（本 EPIC では実装しない、ADR-004 系統の検証機構が整備された際に統合）
