### 追加修正が必要な点

1. **frontmatter `branch_prefix` 優先動作のテスト例が現行 validator と矛盾している**

Issue 本文の完了条件に以下がある:

```text
branch_prefix: hotfix を frontmatter に持つテスト Issue で、label が type:bug でも hotfix/<id> worktree が作成されること
```

しかし現行実装では `kaji_harness/providers/context.py:35-43` の `validate_branch_prefix()` が `LABEL_TO_PREFIX.values()` の既知 prefix のみに制限しており、`hotfix` は許容値ではない。`branch_prefix: hotfix` を持つ Issue は `resolve_issue_context()` 到達前の frontmatter validation で invalid になる想定。

この完了条件は、例えば次のように修正するのが妥当:

```text
branch_prefix: docs を frontmatter に持つテスト Issue で、label が type:bug でも docs/<id> worktree が作成されること
```

または `refactor` / `test` など、`LABEL_TO_PREFIX.values()` に含まれる prefix を使うこと。

### 補足

`.kaji/issues/` 配下に `branch_prefix` frontmatter が 0 件という結論自体は妥当そうだが、本文に書かれている `grep -rn "branch_prefix:" .kaji/issues/` は、Issue 本文やコメント中の説明文にもヒットするため、今後の根拠コマンドとしては厳密ではない。設計書では frontmatter だけを対象にした確認方法に置き換えるとよい。

### 判定

前回 3 件のレビュー指摘への対応は妥当。ただし、完了条件の `branch_prefix: hotfix` 例は現行実装と矛盾するため、着手前に修正してください。
