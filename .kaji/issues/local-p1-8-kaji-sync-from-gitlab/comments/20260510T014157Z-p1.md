1. **`MAX_PAGES` 境界で完了条件を破る**
   `kaji_harness/sync.py:148-168` は `len(payload) == 100` のたびに `page += 1` し、loop 脱出後に `if page > _MAX_PAGES` で即エラーにしています。このため open Issue がちょうど 20,000 件（200 pages * 100 件）ある repo では、201 ページ目を fetch する前に正常終了できず `SyncError` になります。Issue/設計は \"open Issue 全件\" を同期対象にしているため、上限ちょうどの repo で false positive abort するのは要件未達です。`X-Next-Page` を見るか、少なくとも 200 ページ目取得後に \"次ページが必要か\" を判定するテストを追加してください。
