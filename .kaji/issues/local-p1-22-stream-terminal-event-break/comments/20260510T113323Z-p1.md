## 実装完了報告 (TDD)

設計 `draft/design/issue-local-p1-22-stream-terminal-event-break.md` に基づき、TDD で実装を完了。

### 実施内容

- **adapters.py**: `CLIEventAdapter` Protocol に `is_terminal_event` を追加。Claude/Gemini は `type == "result"`、Codex は `type in ("turn.completed", "turn.failed")` を返す。
- **models.py**: `CLIResult` に `terminal_seen: bool = False` を追加（デフォルトで後方互換）。
- **cli.py / stream_and_log**: terminal event 観測時に for loop を break し `terminal_seen=True` を返す。stderr 読み出しは early-break 時の blocking を避けるため EOF 経路でのみ実施。
- **cli.py / _execute_cli_once**: terminal_seen 時は最初に `timer.cancel()` → `process.terminate()` → `wait(5)` → 必要なら `kill()`。stderr は terminate 完了後に補完読み出し。`timed_out and not terminal_seen` ガードで race を抑制。terminal event 観測後の成功/失敗判定は `error_messages` を一次情報とする（SIGTERM 起因の返り値で誤って失敗扱いにしない）。
- **tests/test_adapters.py**: `TestIsTerminalEvent` クラス（10 ケース、Small）。
- **tests/test_cli_streaming_integration.py**: `TestTerminalEventBreak` クラス（6 ケース、Medium）— fd leak 再現、timer race regression、Codex turn.failed terminal、Codex error 非 terminal、no-terminal EOF fallback、最終ガード timeout。
