# 設計修正報告（cycle 1/3）

## 対応済み

### 1. terminal event 後の grace wait 中に既存 timer が発火する race を解消（Must Fix）

- **修正**:
  - `CLIResult` に `terminal_seen: bool = False` を追加（後方互換のためデフォルト False）。
  - `stream_and_log` は terminal event 観測時に `terminal_seen=True` を返すだけにし、後始末（terminate / wait / kill）は呼び出し側に委譲。
  - `_execute_cli_once` で **`result.terminal_seen` が True なら最初に `timer.cancel()` を呼んでから** terminate → wait(5) → kill を行う順序にした。これで grace wait 中に `_kill_process` が発火する race が構造的に消える。
  - `timer.cancel()` が既に発火 race に負けた極端ケースに備え、末尾の `timed_out.is_set()` 判定を `not result.terminal_seen` で短絡するガードを追加。
- **設計書反映箇所**: 「インターフェース」§ `stream_and_log` と `_execute_cli_once`（変更）— race 防止 3 ケース（観測前 timer 発火 / 観測後 cancel 成功 or race 負け / 通常 EOF）を明示。
- **テスト**: 「テスト戦略 Medium」#2 に **timer race regression** を追加。`result` 出力 0.5s 後に sleep 4s、`default_timeout=1` で実行し `StepTimeoutError` ではなく成功で完了することを assert。

### 2. Codex の terminal event を一次情報で確定（Must Fix）

  - `error` event は intermediate 扱い（`tests/test_codex_robustness.py:315-324` の挙動から `error` 単独で session が終わらず後続に `turn.failed` を伴う観測あり。`error` で break すると後続 `turn.failed` の `error.message` が `error_messages` に集約されない既存契約を壊す）。
