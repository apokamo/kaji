## レビュー指摘対応 (cycle 1)

レビュー指摘（Claude/Gemini failure terminal が成功扱いになる回帰）に対応。

### 修正内容

1. **`CLIEventAdapter.is_terminal_failure(event) -> bool` を追加** (`kaji_harness/adapters.py`)
   - **ClaudeAdapter**: `type=="result"` かつ (`is_error is True` または `subtype=="error"`)
   - **CodexAdapter**: `type=="turn.failed"`
   - **GeminiAdapter**: `type=="result"` かつ `status` が `"success"` 以外（None は非失敗扱い）

2. **`CLIResult.terminal_failure: bool = False` を追加** (`kaji_harness/models.py`)

3. **`stream_and_log`**: terminal event 観測時に `adapter.is_terminal_failure(event)` を評価して `terminal_failure` を set。

4. **`_execute_cli_once` の失敗判定を再構成** (`kaji_harness/cli.py`):
   ```
   失敗 := terminal_failure or (returncode > 0) or error_messages
   ```
   我々の SIGTERM 後の `returncode < 0` は失敗根拠にしない（terminal event を真実とする）。
   `returncode > 0`（CLI が自発的に非ゼロ exit）は失敗扱い。

### 追加テスト

**`tests/test_adapters.py::TestIsTerminalFailure`** (Small, 10 ケース):
- Claude: `subtype:"error"` / `is_error:true` で True、`subtype:"success"` / 非 terminal で False
- Codex: `turn.failed` で True、`turn.completed` / `error` で False
- Gemini: `status:"error"` で True、`status:"success"` / 非 terminal で False

**`tests/test_cli_streaming_integration.py::TestTerminalEventBreak`** に 3 件追加 (Medium):
- `test_claude_failure_terminal_raises_cli_execution_error`: Claude `subtype:error / is_error:true` + `exec sleep 30`（fd leak）→ `CLIExecutionError` 発火（修正前は成功 return）
- `test_gemini_failure_terminal_raises_cli_execution_error`: Gemini `status:error` + `exec sleep 30` → `CLIExecutionError` 発火（修正前は成功 return）
- `test_claude_success_terminal_with_self_exit_nonzero_raises`: success terminal でも CLI が自発 `exit 1` した場合は `CLIExecutionError`（returncode>0 の自発 exit を拾う回帰防止）
