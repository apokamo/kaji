### 補足: 一連の BACK ループの根本原因と恒久対策

0010, 0011, 0012 で 3 回 BACK を出した直接原因は「verify-code の numbered comment 不在」だが、その背後にあったのは:

- **環境バグ**: `.kaji/config.local.toml` overlay が `git worktree add` で worktree に転写されず、worktree 配下の `kaji issue *` が `provider.type=github` 既定値で動き、`local-*` 形式 ID を `invalid issue format` で reject していた
- **症状**: codex review-code / verify-code（worktree 配下で稼働）が `kaji issue comment` を実行できず、verdict は run.log に残るが Issue 上の numbered comment は欠落
- **session 持ち越し**: `verify-code` step は `resume: review-code` 指定で codex session を引き継ぐため、修正後の verify-code 再実行でも「post 不可」と判断した過去 turn を踏襲し comment 投稿が再試行されなかった

**恒久対策**:

1. ✅ skill `issue-start` Step 2.6 を新設、`MAIN_REPO/.kaji/config.local.toml` を worktree に symlink (`b8f3010`、main 直コミット)
2. ✅ 既存 worktree (local-pc5090-6) に手動 symlink 適用済み
3. 検証: 修正後 worktree から `kaji config provider-type` → `local`、`kaji issue view local-pc5090-6 --comments` → 正常 JSON

今後 `/issue-start` から作成される worktree は最初から overlay を共有するため、本症状は構造的に再発しない。
