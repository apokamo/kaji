## 実装完了報告 (TDD)

設計書 `draft/design/issue-local-pc5090-6-kaji-issue-pr-gitlab-passthrough.md` に基づき、TDD で実装を完了しました。

### 実施内容

- **normalize_id 拡張**: `gl:N` 形式追加、`ResolvedKind` Literal に `\"gitlab\"` 追加。cross-provider (`gh:N` × gitlab / `gl:N` × github) は対称な ValueError で拒否
- **provider PR helper**: `_GitLabPrShape`（純粋な GL→GH JSON 変換）と `resolve_mr_iid_from_branch` / `list_pr_review_comments` / `list_pr_reviews`（notes + approvals の marker join）/ `reply_to_pr_comment` / `get_mr_approval_state` を `gitlab.py` に追加
- **`kaji issue` GitLab dispatcher**: sub 名/flag 写像（`edit→update`、`comment→note`、`--body→--description`/`--message`）、`view`/`list` の `--json/--jq` は既存 `view_issue`/`list_issues` を再利用して GitHub 互換 dict から jq 適用
- **`kaji pr` GitLab dispatcher**: Tier B（create/view/list/merge/comment/review）+ Tier A（review-comments/reviews/reply-to-comment）。`--base→--target-branch`、`--head→--source-branch`、`--squash/--rebase` 拒否、branch→IID 解決、kaji-review marker 注入による note→approve/revoke シーケンス、未対応 sub の rc=2 reject
- **skill 修正**: `i-pr` / `issue-close` の `provider_type` 受理を `gitlab` まで拡張、`i-pr` の PR URL regex を gitlab.com の `/-/merge_requests/<N>` 形式にも対応
