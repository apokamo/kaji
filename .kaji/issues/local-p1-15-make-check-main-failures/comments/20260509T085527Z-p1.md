## 指摘事項 (Must Fix)

- [ ] **原因Aの config 隔離 fixture の scope を修正する**: 設計書は「各 affected class に class-scoped autouse fixture を追加し、`monkeypatch.setattr` で `_load_config_for_dispatch` を stub」としていますが、pytest 標準の `monkeypatch` fixture は function-scoped です。class-scoped fixture から要求すると `ScopeMismatch` でセットアップ失敗になります。
  - **具体的根拠**: 最小再現として `@pytest.fixture(scope="class", autouse=True) def fx(monkeypatch): ...` を実行すると `ScopeMismatch: You tried to access the function scoped fixture monkeypatch with a class scoped request object.` で落ちることを確認しました。
  - **影響**: 原因Aの実装前提が崩れるため、設計段階完了条件「テストでの config 隔離方式が決定されている」を満たしていません。
  - **修正の方向性**: `scope` を付けない function-scoped autouse fixture にする、`pytest.MonkeyPatch()` を手動生成して `yield` / `undo()` する class-scoped fixture にする、または既存テストを per-test `with patch(...)` へ寄せる、のいずれかに設計を落とし直してください。
