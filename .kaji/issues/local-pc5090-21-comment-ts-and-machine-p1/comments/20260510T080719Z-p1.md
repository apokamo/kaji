# レビュー指摘への対応報告

## 対応済み

### Must Fix 1: migration script の repo_main 解決を修正

`scripts/migrate_comment_filenames_and_machine.py` を改修。

- `_resolve_repo_main(repo)` を追加し、`--repo-main` 省略時に以下の順で main repo を自動解決:
  1. `.kaji/config.local.toml` が symlink ならその実体の親の親を main repo として採用
  2. `git rev-parse --path-format=absolute --git-common-dir` の親（worktree → main repo の git metadata 経由）
  3. 解決不能なら `repo` 自身（main repo として実行された場合）
- 「counter file が見つからない」「config の machine_id が `pc5090` でも `p1` でもない」を warning から **`SystemExit` (fail-fast)** に格上げ。完了条件 (`.kaji/counters/p1.txt` 存在 / `pc5090.txt` 削除 / config 切替) を保証するため。

検証:
```
$ python scripts/migrate_comment_filenames_and_machine.py --phase=config --dry-run
[config] repo_main=/home/aki/dev/kaji/main
  counter already at .kaji/counters/p1.txt (idempotent)
  config already migrated (idempotent): /home/aki/dev/kaji/main/.kaji/config.local.toml
```

worktree から `--repo-main` 省略実行で main repo (`/home/aki/dev/kaji/main`) を自動検出することを確認。
