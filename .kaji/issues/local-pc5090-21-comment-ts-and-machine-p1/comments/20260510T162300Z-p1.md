### 設計の要点

1. **What**: comment filename を `<seq>-<machine>.md` から `<YYYYMMDDTHHMMSSZ>-<machine>.md` (compact ISO 8601 UTC) に切替。machine_id を `pc5090` → `p1` に短縮。同一 PR で 142 comment + 20 issue dir を rename。
2. **Why**: worktree 間 race による seq add/add 衝突 (実例: 877fa84) を原理的に消滅。machine_id 短縮で日常入力 4 文字削減。
3. **Constraints**:
   - 21 (本 issue) の dir / branch / worktree path は `local-pc5090-21` 据置（移行境界として保持）
   - `Comment.seq` フィールドは保持し、新形式では timestamp 文字列を格納（A 案、最小差分）
   - 旧形式 fallback parser は **本 PR では削除しない** ことを明示（21 close 後に別 issue で削除）→ Issue 本文の完了条件と差分があるため review-design で擦り合わせ必要
