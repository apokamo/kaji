# レビュー指摘への対応報告

レビューありがとうございます。3 件の Must Fix について以下の通り検討・対応しました。

## 対応済み

### Must Fix #2: review --approve の順序契約を pin

- 修正内容: `tests/test_large_gitlab/test_pr_review_contract.py:188-220` (Independent observation 3 を追加)
- 設計どおり `note.created_at <= approvals.updated_at` を観測。GitLab API の timestamp が秒精度のため、同秒で揃った場合は `warnings.warn` に降格して flake 化を防止（contract は「両立」を必達、「順序」は観測可能なときのみ assert という二段構え）。`note.created_at > approvals.updated_at` の真の順序逆転は AssertionError で失敗。timestamp field 自体が欠けた場合は shape regression として `pytest.fail` で surface。
- ISO8601 paser (`_parse_iso8601`) はトレイリング `Z` を `+00:00` に正規化してから `datetime.fromisoformat` に渡す。

### Must Fix #3: diff comment の path / line を実測で固定

- 修正内容: `tests/test_large_gitlab/test_review_shape.py:40-200`（helper を positional discussion 化）+ `test_review_comments_have_github_compatible_shape` 本体に `path` / `line` assert を追加
- helper `_open_mr_with_diff_thread` を non-positional から **positional (text position)** に変更。MR 作成後に `diff_refs` を取得（必要に応じて `glab api projects/<encoded>/merge_requests/<iid>` で refetch）し、`position[base_sha]` / `position[head_sha]` / `position[start_sha]` / `position[new_path]` / `position[new_line]` を指定して discussion を投稿。
- `test_review_comments_have_github_compatible_shape` は `id` / `user` / `body` / `path` / `line` / `created_at` / `in_reply_to_id` の完全 shape を確認し、`item["path"] == new_path` / `item["line"] == new_line` を実測で固定。provider が `glab api .../discussions → GitHub subset` 変換で `path` / `line` を落とす回帰を捕捉できる。
- helper return tuple も `(iid, branch, discussion_id, new_path, new_line)` に拡張し、3 テスト全箇所の呼び出しを更新。docstring も positional 前提に書き直し。
