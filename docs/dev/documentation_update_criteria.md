# ドキュメント更新基準

コード変更に伴うドキュメント更新の要否を判断するフレームワーク。
`/issue-doc-check` で使用する。

## 判断フロー

```
変更内容を確認
  ├── CLI の引数・出力が変わった → cli-guides/ を更新
  ├── アーキテクチャが変わった → ARCHITECTURE.md を更新
  ├── 新しい技術選定をした → docs/adr/ に ADR を追加
  ├── 開発手順が変わった → docs/dev/ を更新
  ├── 規約が変わった → CLAUDE.md を更新
  └── 上記いずれにも該当しない → 更新不要
```

## 更新対象ドキュメント一覧

| ドキュメント | 更新トリガー |
|-------------|-------------|
| `CLAUDE.md` | 規約・コマンド・ツール設定の変更 |
| `docs/ARCHITECTURE.md` | モジュール構成・依存関係・データフローの変更 |
| `docs/adr/` | 重要な技術選定・方針決定 |
| `docs/cli-guides/` | CLI コマンドの追加・変更・削除 |
| `docs/dev/` | 開発ワークフロー・手順・規約の変更 |
| `docs/guides/` | 運用ガイド・手順の変更 |

## 更新不要のケース

- 内部リファクタリングで外部インターフェースが変わらない場合
- テスト追加のみの場合
- 既存ドキュメントが既に正確な場合

## 設計書の「影響ドキュメント」テーブルとの関係

設計書に記載された「影響ドキュメント」テーブルは設計時点の予測。
`/issue-doc-check` では実装後の実際の変更内容に基づいて再判定する。
