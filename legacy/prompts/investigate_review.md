# INVESTIGATE Review Prompt

Issue ${issue_url} の `## Bugfix agent INVESTIGATE` セクションをレビューしてください。

## タスク

1. `gh issue view` で最新の Issue 本文を取得
2. INVESTIGATE ステートの必須アウトプットを確認
3. DETAIL_DESIGN ステートに移行可能か判定

## 完了条件チェックリスト

| # | 項目 | 確認内容 |
|---|------|----------|
| 1 | **再現手順と証跡** | 実行したステップと結果が記載され、証跡ファイルへの参照がある |
| 2 | **期待値との差** | 期待する挙動との乖離が明確に記載されている |
| 3 | **原因仮説** | 可能性のある原因が根拠（ログや該当コード）と共に列挙されている |
| 4 | **補足情報** | 設計に役立つ追加情報がある（なくても可） |

## 禁止事項

**次ステート以降の責務を新規に実行しない**
- 例：設計作成、実装案の提示

**既に完了した責務を再実行しない**
- 例：再現を最初からやり直す、調査を再実行

※ 記載内容の検証（整合性チェック、矛盾指摘）は**許可**されています。

## 出力形式

```markdown
### INVESTIGATE Review Result

#### 検証内容
- <実施した検証と結果を具体的に記載>

#### チェックリスト
- 再現手順と証跡: <OK/NG + 具体的根拠>
- 期待値との差: <OK/NG + 具体的根拠>
- 原因仮説: <OK/NG + 具体的根拠>
- 補足情報: <OK/NG or N/A>

## VERDICT
- Result: PASS | RETRY
- Reason: <判定理由>
- Evidence: <具体的な判断根拠（Issue本文の該当箇所、確認した内容）>
- Suggestion: <RETRY時: 具体的な追加調査指示>
```

## 判定ガイドライン

| 状況 | VERDICT | 次のステート |
|------|---------|-------------|
| 必須項目が揃い、矛盾・不足がない | PASS | DETAIL_DESIGN |
| 不足・矛盾があり再調査が必要 | RETRY | INVESTIGATE |

## レポート方法

1. `gh issue comment` で Issue にコメント投稿（VERDICT判定）
2. **PASS判定時のみ**: 共通ルールに従い `gh issue edit` で Issue 本文を更新
