# PR Prepare

> [!WARNING]
> **非推奨**: このコマンドは `/issue-pr` に統合されました。
> Issue駆動開発ワークフローでは `/issue-pr` を使用してください。
> このコマンドは互換性のため一定期間残されますが、将来削除される予定です。

PR作成前にコミット履歴を整理し、リモートにプッシュしてPRを作成します。

## いつ使うか

| タイミング | このスキルを使用 |
|-----------|-----------------|
| Issue駆動ワークフロー | ❌ `/issue-pr` を使用 |
| worktree外での単発PR | ⚠️ 使用可（非推奨） |

## 前提条件

- `git absorb` がインストール済み
- 作業ブランチで全ての変更がコミット済み

## 実行手順

### Step 1: 未コミットの変更確認

```bash
git status
```

未コミットの変更がある場合は先にコミットしてください。

### Step 2: コミット履歴の整理

```bash
git absorb --and-rebase
```

fixup対象がない場合は何も起きません（正常）。

### Step 3: リモートにプッシュ

```bash
git push -u origin HEAD
```

### Step 4: PR作成

```bash
gh pr create --fill
```

`--fill` は最初のコミットメッセージからタイトルと説明を自動生成します。
カスタマイズが必要な場合は対話モードで:

```bash
gh pr create
```

### Step 5: 完了報告

以下の形式で報告してください:

```
## PR準備完了

| 項目 | 状態 |
|------|------|
| コミット整理 | git absorb 実行済み |
| プッシュ | 完了 |
| PR | 作成済み |

PR URL: [URL]
```
