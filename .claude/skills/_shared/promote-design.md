# 設計書の昇格手順（共通）

draft 設計書から恒久ドキュメント（`docs/adr/` 等）への昇格が必要な場合の手順。

## いつ昇格するか

| 条件 | 昇格先 |
|------|--------|
| アーキテクチャに関わる技術選定・方針決定 | `docs/adr/` に ADR として |
| 汎用的な開発ガイドライン | `docs/dev/` にガイドとして |
| 上記以外 | 昇格不要（Issue 本文へのアーカイブで十分） |

## 手順

1. **昇格対象の判断**: 設計書の内容がアーキテクチャ決定や汎用ガイドラインに該当するか確認
2. **昇格先ファイルの作成**: `docs/adr/NNNN-title.md` または `docs/dev/xxx.md` を作成
3. **内容の変換**: draft 設計書から恒久ドキュメントに適した形式に変換（Issue 固有の文脈を除去）
4. **設計書からの参照**: 元の draft 設計書に昇格先ドキュメントへのリンクを追記
5. **コミット**: 昇格先ドキュメントと設計書の更新を一緒にコミット

## 注意事項

- 昇格は任意。すべての設計書が昇格対象ではない
- 昇格先ドキュメントは draft 設計書のコピーではなく、恒久ドキュメントとして独立した内容にする
- Issue 番号や worktree パス等の一時的な情報は含めない
